Outbush AI build plan

A local field guide for the last bar of signal.

Outbush AI is a Gradio app for Australian bushwalkers: useful on a phone, served from a Raspberry Pi 5 hotspot, grounded in local knowledge, and deliberately cautious whenever the stakes are first aid, wildlife, plants, weather, or bush tucker.

Summary

Build one app that runs fully offline on the Pi and also has a Hugging Face Space mirror for judges. Cloud credits are used for preparation work: fine-tuning, quantization, benchmarking, publishing, and demo artifacts. Runtime field use should not depend on cloud APIs.

Primary audience

Australian bushwalkers using a mobile phone after service drops out, especially groups who need practical reminders, hazard triage, checklist support, and plain-language emergency guidance.

Product stance

Helpful, calm, and specific without pretending certainty. The app can suggest possibilities and next actions, but it never outranks situational awareness, official advice, emergency services, or trained first aid.

Product Ideas And Exit Criteria

Each feature has a concrete exit test so the final build can be checked in the field, not just admired in a README.

Field Chat / Ask Outbush

Mobile-first chat for wildlife, plants, survival, first aid, gear, and weather questions using the local LLM plus curated retrieval.

Exit: works offline from a phone on Pi Wi-Fi; answers include source labels, uncertainty, and practical next steps.

Photo "What Is This?"

Phone photo upload for candidate plant, animal, fungus, or cloud identification.

Exit: dangerous-species and mushroom test images trigger strong warnings; the app never says a wild item is safe to eat or touch.

Danger Mode

Quick cards for Australian snakes, spiders, dangerous animals, stinging plants, ticks, and mushrooms.

Exit: hazards map to clear "stop / move away / call 000 or Poisons 13 11 26 where relevant" guidance.

First Aid And Survival

Concise emergency flows for snake bite, funnel-web or mouse spider bite, bleeding, heat illness, dehydration, getting lost, exposure, and signalling.

Exit: content is grounded in official Australian sources and passes a safety regression suite.

Bush Tucker With Guardrails

Educational foraging information, not consumption approval. Mushrooms always warn "do not eat wild mushrooms."

Exit: all edible-adjacent answers include expert-verification warnings and no photo-only edible confirmation.

Checklist And Trip Intentions

Pre-walk checklist, trip plan fields, emergency contacts, PLB/phone reminders, water, food, map, and weather prompts.

Exit: user can complete the checklist on mobile and export or share text before leaving service.

Weather And Cloud Read

BoM climate averages by broad Australian region plus educational cloud-photo interpretation.

Exit: app clearly distinguishes historical climate averages, cloud observation, and live forecasts the user must check before departure.

Offline Encyclopedia

Curated Australia-relevant knowledge pack with selected Wikipedia or Kiwix content plus official safety pages indexed locally.

Exit: searchable offline answers work without internet and cite the local source title and retrieval date.

Pi Wi-Fi Hotspot

Pi broadcasts Outbush-AI, serves the app at http://outbush.local, and exposes a QR/on-device landing page.

Exit: multiple phones can connect in airplane mode and use the app with no external network.

Safety Policy

Outbush AI should be maximally useful and maximally honest. The app can help someone think clearly under pressure, but it must never invent certainty in a high-risk setting.

Hard rules

  • Never confirm a wild plant, fungus, or animal is safe to eat or touch from a photo.
  • Always warn against eating wild mushrooms.
  • Escalate life-threatening symptoms to Triple Zero (000), or 112 if appropriate from a mobile.
  • For poisoning, bites, stings, or suspected ingestion, point to Poisons Information Centre 13 11 26 unless immediate ambulance care is needed.
  • Separate historical climate averages and cloud signs from live weather forecasts.

Answer format

Every high-stakes answer should include: uncertainty level, what to do now, what not to do, when to escalate, and local source labels. This makes the model useful without turning it into a fake authority.

Implementation Architecture

Standalone plan artifact

Create this HTML document first so the product, scope, safety posture, and exit criteria are human-readable and shareable.

Custom Gradio app

Use gradio.Server with custom HTML, CSS, and JavaScript served at /. Register queued Gradio API endpoints plus simple local REST routes for the offline frontend.

Local retrieval

Use SQLite FTS5 for a lightweight offline index over official safety pages, selected field notes, checklist content, climate summaries, and compressed encyclopedia extracts.

Runtime model path

Serve the text model through llama.cpp on the Pi. Keep deterministic safety fallbacks in the app so the interface remains useful when the model is loading or unavailable.

Submission packaging

Publish the Hugging Face Space, model, dataset, demo video, social post, field notes, and optional agent trace so the judging evidence is explicit.

Models, Data, And Tooling

Default model track

Use a small GGUF text model such as Qwen2.5-3B-Instruct Q4_K_M, LoRA-tuned for Outbush answer style, safety wrappers, and source-aware field responses. Publish the tuned model to Hugging Face.

Use SmolVLM2-2.2B-Instruct-GGUF Q4_K_M as the first vision candidate, then benchmark on Pi and fall back if latency or memory is poor.

Credits and services

Use the installed Hugging Face CLI and Modal for model training, evaluation, conversion, quantization, and publishing. Field runtime remains local: no external calls are required after the Pi is prepared.

Raspberry Pi Deployment

The Pi is the product, not a novelty host. It must feel like a field appliance: predictable boot, reliable Wi-Fi, readable phone UI, and no internet dependency.

Services

llama-server, Outbush Gradio app, optional kiwix-serve, hotspot, mDNS, and a health endpoint.

Hardware

Raspberry Pi 5 8 GB, cooling, stable battery pack, enclosure, SD or NVMe storage, and a simple recovery/backup image.

Field access

SSID Outbush-AI, local URL http://outbush.local, QR code, and a fallback IP address sticker.

Tests And Completion Goals

Build checks

  • the_plan.html opens locally, is mobile-readable, includes exit criteria, and links sources.
  • Local app runs on desktop with no missing routes; endpoints return structured safe responses.
  • Safety tests cover emergency prompts, mushroom warnings, no photo-only edible claims, source citation, and escalation.
  • Performance target: Pi under roughly 7 GB RAM, short text answer under about 15 seconds, photo analysis under about 60 seconds after model load.

Field checks

  • Phone joins Pi Wi-Fi in airplane mode and uses chat, photo, checklist, and weather tools.
  • HF Space is public, Gradio-based, and links model/data repos plus demo materials.
  • You and friends complete a short phone-on-Pi field trial and capture feedback for Field Notes.

Risks And Missing Pieces

Fresh conditions

Offline mode cannot know current fire bans, park closures, flood warnings, track closures, or live weather unless the user pre-syncs before departure.

Privacy

Uploaded photos should be processed locally, not retained by default, and excluded from ordinary logs.

Cultural care

Bush tucker content must be careful and sourced. Do not present Indigenous knowledge as exhaustive, generic, or detached from context.
