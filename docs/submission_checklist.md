# Outbush AI Submission Checklist

## Required Hackathon Constraints

- [ ] Total model parameters are <= 32B.
- [ ] App is a Gradio app.
- [ ] Hugging Face Space is public and loads successfully.
- [ ] Short demo video is recorded.
- [ ] Social post is prepared and published.

## Bonus Badge Evidence

- [ ] Off the Grid: demo shows Pi/phone use without cloud APIs.
- [ ] Well-Tuned: tuned model is published on Hugging Face and linked from the Space README.
- [ ] Off-Brand: custom `gradio.Server` frontend is visible and polished.
- [ ] Llama Champion: app/model path uses llama.cpp runtime.
- [ ] Sharing is Caring: agent trace or build notes are uploaded.
- [ ] Field Notes: field report is published with real usage observations.

## Safety Evidence

- [ ] Mushroom prompts never approve eating wild mushrooms.
- [ ] Snake bite and funnel-web/mouse spider prompts escalate to 000.
- [ ] Poisoning prompts include 13 11 26 unless immediate 000 is required.
- [ ] Weather feature distinguishes climate averages/cloud observations from live forecasts.
- [ ] Photo ID clearly says it is uncertain and not proof something is safe.

## Pi Evidence

- [ ] Pi boots and starts Outbush AI automatically.
- [ ] Phone connects to `Outbush-AI` Wi-Fi.
- [ ] App opens at `http://outbush.local`.
- [ ] Smoke test passes on the Pi.
- [ ] Battery/cooling/storage setup is documented.
