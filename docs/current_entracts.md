## Methodology

Scores are heuristic triage scores out of 100, based only on public, visible evidence:
Space card metadata, tags, README text, model references, file names, likes, and repository
structure. They are not judge predictions. The rubric weights favor the exact published
criteria: concrete real-user problem and small-model fit for Backyard AI; delight,
originality, load-bearing AI, and polish for Thousand Token Wood.

## Interpreting The Scores

- **80+**: strong visible fit; worth direct human review.
- **65-79**: plausible contender, usually missing one public proof point.
- **50-64**: interesting but weaker or less clearly aligned from public metadata.
- **Below 50**: may still be good, but the public evidence does not match the chapter rubric strongly.

## Caveats

- The Hugging Face org page was described as having 426 Spaces, but the logged-out API
  returned 393 public Spaces during this run. Private, deleted, or unlisted
  Spaces are not included.
- README/social/demo links are detected from text. The script does not watch every demo
  video or execute every app.
- The scoring favors explicit evidence. A subtle but excellent app can rank lower if its
  README/card does not state who used it, what model it uses, or why the AI is central.
